The hX+jF Demo

Stellar Help Exchange (hX) is an equitable cloud-based marketplace. The demo runs on Stellar testnet in 3 steps:
  • reset hX on testnet;
  • setup hX demo data; and
  • run the demo.
Each step has one or more jobs you request from this browser. Cloudflare Job Fair (jF) matches your job requests with Job Agents that offer the jobs you need. Without the jF functionality, the demo would not run, so this is actually a double demo, hX+jF.

This test has 2 parts. First, 2 Job Agents, each one offering echo job, connect to jF before your echo job request is sent from this browser. Then, one of them is being matched and, if your handshake with it is successful, runs the echo job for you. The second agent is still hanging, so you send another echo job request to jF and, having been matched with it, get another echo job done for you - assuming, of course, that your handshake with the agent was successful.

The second part of the test starts when you send a third job request. The request hangs there for a little while until the 2 Job Agents connect to jF again. Then, one of them is being matched and, if your handshake with it is successful, runs the echo job for you. The second agent is still hanging, so you send your final job request to jF and, having been matched with it, get another echo job done for you - assuming, of course, that your handshake with the agent was successful.

All messages are signed by the sender and verified on the edge and by the recipient. It is worth noting here that jF Job Agents in qa MUST use mTLS.

On the edge, each jF participant - a Job Agent or a Job Client like this browser - is represented by an open websocket connection and the associated attachment. The attachment keeps the breakpoint of the State and Context objects for the connection.
