The TM High Availability Test

Stellar Testnet Monitor (TM) is used to start demo Job Agents whenever the demo is requested. If another demo is in progress, the request waits in a queue. The TM implementation is based on the Stellar TESTNET orderbook. The TM agent places a sell order for the Monitor Asset (MA). You place a buy order for the same amount of MA. When your order executes, the demo starts for you. When the demo completes (or when the timeout on the TM agent expires), another sell order for the same amount of MA is being placed by the agent. Now the TM is either waiting for someone else to place a buy order, or executes an order already placed. Basically, the TM is a cloud-based semaphore: 
             +-------+  +----+     o
             | Agent |  | TM |    -|-
             +-------+  +----+    / \...
                 |        |        |
                 | release|        |
                 |------->|        |
                 |  A     |acquire |
                 |  |     |<-------|
    demo starts ||| |     
                ||| |
                ||| |
 demo completes |||-/
    (or timeout  |
       expires)

When your demo request is granted, the hX+jF demo page pops up from this page. IMPORTANT: to see the demo, you MUST allow this popup.
